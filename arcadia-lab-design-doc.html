Panoramica del sistema

L'area personale di Arcadia Lab. è suddivisa in due macro-aree distinte: il Pannello Admin (riservato alla titolare dello studio) e l'Area Personale Utente (riservata agli iscritti). L'accesso è protetto da autenticazione Supabase con separazione dei ruoli basata sull'email.

Sistema di autenticazione

L'autenticazione è gestita interamente da Supabase Auth. La pagina /auth espone tre modalità: Login, Registrazione e Recupero password. Il contesto AuthContext rende la sessione disponibile a tutta l'applicazione.

Flusso di autenticazione

01Visita /authPagina pubblica
02Login / RegistrazioneEmail + Password
03Supabase AuthVerifica credenziali
04Sessione JWTToken salvato in cookie
05Redirect /dashboardRuolo rilevato dall'email
🔐
Pagina /auth — tre modalità
LoginEmail + password. Errori tradotti in italiano. Reindirizza a /dashboard al successo.
RegistrazioneEmail + password + conferma. Validazione lunghezza minima 6 caratteri. Email di conferma automatica da Supabase.
Recupero passwordInserisci email → ricezione link di reset → nuova password. Link scade dopo 24h.
ProtectedRouteHOC che verifica la sessione. Se non loggato, redirect automatico a /auth.
⚠️
Il sistema dei ruoli (admin vs utente) è attualmente basato su una whitelist di email hardcoded nel frontend. Per produzione si consiglia di spostare il controllo del ruolo su Supabase Row Level Security (RLS) o nella tabella profiles.

Schema PostgreSQL consigliato per gestire utenti, abbonamenti, lezioni e prenotazioni. Da creare nel pannello Supabase → SQL Editor.

Tabella: profiles

avatar_urltextURL immagine profilo (opzionale)
notifiche_emailjsonbPreferenze notifiche: {lezioni, scadenza, promo}
created_attimestamptzData creazione — default: now()

Tabella: subscriptions

CampoTipoNote
id PKuuidGenerato automaticamente
user_iduuidFK → profiles.id
pianotextValori: mensile | trimestrale | annuale | singola
lezioni_totaliintegerEs: 12 per mensile, 36 per trimestrale
lezioni_usateintegerIncrementato a ogni presenza registrata
data_iniziodateData di attivazione abbonamento
data_scadenzadateData di scadenza abbonamento

prezzo_pagatonumericImporto in euro al momento dell'acquisto
created_attimestamptzData creazione record

Tabella: lessons

CampoTipoNote
id PKuuidGenerato automaticamente
titolotextEs: "Hatha Yoga", "Vinyasa Flow"
data_oratimestamptzData e ora della lezione
durata_minintegerDurata in minuti (es: 60, 90)
posti_totaliintegerNumero massimo di partecipanti
posti_disponibiliintegerAggiornato a ogni prenotazione/disdetta

Tabella: bookings

CampoTipoNote
id PKuuidGenerato automaticamente
user_iduuidFK → profiles.id
lesson_iduuidFK → lessons.id
subscription_iduuidFK → subscriptions.id (abbonamento usato)
statotextValori: confermata | in-attesa | cancellata
presenzabooleanSegnata dall'admin dopo la lezione
created_attimestamptzData prenotazione
✅
Tutte le tabelle devono avere Row Level Security (RLS) abilitata. Gli utenti possono leggere solo i propri dati (user_id = auth.uid()). L'admin può leggere tutto tramite una policy separata basata sul ruolo.

13 — Tutti

Impostazioni account

La sezione impostazioni è condivisa tra admin e utenti. L'admin vede in aggiunta il campo Bio/Descrizione studio. Tutte le azioni di modifica chiamano direttamente Supabase Auth API.
Info accountEmail, data creazione account, badge ruolo (Admin/Utente)
Dati personaliNome, cognome, telefono, bio (solo admin). Salva su Supabase user_metadata.
Cambio passwordNuova password + conferma. Validazione minimo 6 caratteri. Salva tramite supabase.auth.updateUser().
Notifiche emailToggle per: promemoria lezioni, scadenza abbonamento, offerte e novità.
Privacy & GDPREsporta i tuoi dati (download JSON), gestisci consenso marketing.
Zona pericolosaEliminazione account con conferma. Irreversibile. Triggera signOut() dopo eliminazione.
